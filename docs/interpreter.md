# Celed Interpreter Documentation

The Interpreter is the third stage of the Celed compiler pipeline. It takes the Abstract Syntax Tree (AST) generated by the Parser and actually executes the code.

## Core Concepts

### Objects (`src/object.ts`)
In Celed, everything is an Object. When we evaluate `5`, we get an `Integer` object. When we define a function, we get a `FunctionObj` which holds the function's parameters, body, and scope.

### Environment (`src/environment.ts`)
The Environment is the memory of our language. When you write `multiplier := 5`, the Interpreter stores the `Integer(5)` object in the Environment under the name `"multiplier"`. Functions get their own sub-environments, allowing for local variables!

### Evaluator (`src/evaluator.ts`)
The Evaluator walks through the AST nodes one by one. 
- If it sees a `+`, it adds the left and right values together.
- If it sees an `IfExpression`, it evaluates the condition and executes the correct block.
- If it sees the `|>` operator, it evaluates the left side and injects it as an argument into the function on the right side.

## No Comments Rule
In adherence to the design rules, `src/evaluator.ts`, `src/object.ts`, and `src/environment.ts` have been written completely devoid of code comments. The logic is kept functional and modular for maximum readability.
