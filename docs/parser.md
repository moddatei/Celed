# Celed Parser Documentation

The Parser is the second stage of the Celed compiler pipeline. It takes the Tokens generated by the Lexer and organizes them into an Abstract Syntax Tree (AST).

## Purpose
While the Lexer turns characters into words, the Parser turns those words into logical sentences. It ensures that the syntax of Celed makes sense. For instance, it ensures that `greeting := "Welcome"` is structured logically with an identifier on the left, an assignment operator in the middle, and a value on the right.

## AST Nodes
We define several nodes in `src/ast.ts` that represent structural parts of Celed code:
- `Program`: The root of the tree.
- `AssignStatement`: Handles `name := value`.
- `FunctionLiteral`: Handles `(...) => { ... }`.
- `PipelineExpression`: Handles the `|>` data flow operator.

## Implementation Details
We use a **Pratt Parser** (a Top-Down Operator Precedence parser). This handles operators like `+` and `*` elegantly, ensuring that `10 + 5 * 2` is parsed as `10 + (5 * 2)` rather than `(10 + 5) * 2`.

Like the Lexer, the Parser (`src/parser.ts`) has been written strictly without comments.
